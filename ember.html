ember browser

a lightweight desktop browser built with c++, webkitgtk, and gtk4. basically made this cuz i was bored and wanted a clean browser that actually does what i want

what it does

  • tabs - obviously. multiple tabs like any normal browser
  • navigation - back, forward, reload, home. the usual stuff
  • url bar - type urls or search directly, works how you'd expect
  • bookmarks - saves your fav sites in json
  • history - sqlite db tracks where you've been
  • settings - customize homepage, search engine, font size, ad blocking
  • downloads - keeps track of your downloads
  • ad blocker - blocks ads (optional but recommended ngl)
  • keyboard shortcuts - ctrl+t, ctrl+w, ctrl+r etc. if you know you know

how it's built

split into modular components so it's not spaghetti code:

  • src/ui/ - all the ui stuff (main window, nav bar, tabs)
  • src/browser/ - browser engine and webview wrappers
  • src/tabs/ - tab management
  • src/settings/ - settings manager
  • src/history/ - history tracking with sqlite
  • src/bookmarks/ - bookmark storage
  • src/downloads/ - download manager
  • src/adblock/ - ad blocking system

what you need

ubuntu/debian

sudo apt-get install build-essential cmake pkg-config
sudo apt-get install libwebkit2gtk-4.1-dev libgtk-4-dev libsqlite3-dev libjson-glib-dev

fedora

sudo dnf install gcc-c++ cmake pkg-config
sudo dnf install webkit2gtk4.1-devel gtk4-devel sqlite-devel json-glib-devel

arch btw

sudo pacman -S base-devel cmake pkg-config
sudo pacman -S webkit2gtk gtk4 sqlite json-glib

requirements

  • cmake 3.16+
  • c++17 compiler (gcc 7+ or clang 5+)
  • webkitgtk 2.40+
  • gtk4 4.0+
  • sqlite3
  • pkg-config

building

  1. go to the project folder: cd ember
  2. make a build directory: mkdir build && cd build
  3. configure: cmake ..
  4. build it: ninja
  5. test if you want: ctest

installing

system-wide (shows up in your app launcher)

cd build
sudo ninja install
sudo gtk-update-icon-cache /usr/share/icons/hicolor

installs to:

  • /usr/bin/ember - the actual app
  • /usr/share/applications/ - desktop launcher
  • /usr/share/icons/hicolor/ - icon

after this it'll show up in gnome or whatever de you're using

local install (no sudo)

cd build
cmake --install . --prefix ~/.local
gtk-update-icon-cache ~/.local/share/icons/hicolor

make sure ~/.local/bin is in your path tho

running

after building: ./ember

or from build directory: ./build/ember

if you installed it just run: ember

or find it in your app launcher

config

ember saves stuff to ~/.config/ember/:

  • settings.conf - your settings
  • history.db - browsing history
  • bookmarks.json - your bookmarks

keyboard shortcuts

  • ctrl+t - new tab
  • ctrl+w - close tab
  • ctrl+r - reload
  • alt+home - go home
  • ctrl+q - quit

settings

open settings from the menu to change:

  • homepage - what loads when you open a tab
  • search engine - google, duckduckgo, or bing
  • font size - make text bigger or smaller (0.5x to 3.0x)
  • ad blocker - turn it on or off

ad blocking

uses a domain blacklist to block:

  • ad servers
  • tracking scripts
  • sketchy domains

you can edit src/adblock/filter_list.hpp to customize what gets blocked

troubleshooting

build failing?

  1. check you installed all the dependencies
  2. make sure pkg-config can find them: 
    pkg-config --modversion webkit2gtk-4.1
pkg-config --modversion gtk4

won't start?

  1. check webkitgtk and gtk4 are installed
  2. make sure you can write to ~/.config/ember/
  3. look at the error messages lol

pages not loading?

  • check your internet connection fr
  • some sites need javascript enabled
  • make sure the url is valid
