Shell Scripting Patterns I Keep Reusing

 2026-02-10 ·  1 min read · #bash #shell #scripting #linux

A collection of shell patterns I find myself reaching for again and again — argument parsing, temp files, logging, and a few traps.

Not a tutorial — just patterns I reach for repeatedly and keep having to re-look up. Writing them down once.

🔗Safe script header

#!/usr/bin/env bash
set -euo pipefail
IFS=$'\n\t'

-e exits on error, -u treats unset variables as errors, -o pipefail catches failures in pipelines. IFS avoids word-splitting surprises.

🔗Temp files with cleanup

tmpfile=$(mktemp)
trap 'rm -f "$tmpfile"' EXIT

# use $tmpfile safely
echo "data" > "$tmpfile"

The trap on EXIT runs even on errors or signals. Never litter /tmp.

🔗Argument parsing with getopts

usage() { echo "Usage: $0 [-v] [-o outfile] <input>"; exit 1; }

verbose=0
outfile=""

while getopts ":vo:" opt; do
  case $opt in
    v) verbose=1 ;;
    o) outfile="$OPTARG" ;;
    :) echo "Option -$OPTARG requires an argument." >&2; usage ;;
    \?) echo "Unknown option: -$OPTARG" >&2; usage ;;
  esac
done
shift $((OPTIND - 1))

[[ $# -lt 1 ]] && usage
input="$1"

🔗Logging to stderr

log()  { echo "[$(date '+%T')] $*" >&2; }
warn() { echo "[$(date '+%T')] WARN: $*" >&2; }
die()  { echo "[$(date '+%T')] ERROR: $*" >&2; exit 1; }

log "Starting up"
[[ -f "$input" ]] || die "Input file not found: $input"

🔗Check for required commands

require() {
  for cmd in "$@"; do
    command -v "$cmd" &>/dev/null || die "Required command not found: $cmd"
  done
}

require curl jq git

🔗Process a file line by line

# correct: preserves leading/trailing whitespace, handles no trailing newline
while IFS= read -r line || [[ -n "$line" ]]; do
  echo ">> $line"
done < "$input"

The || [[ -n "$line" ]] handles files that don't end in a newline.

These six patterns cover maybe 80% of the scripts I write. The rest is just domain logic.
