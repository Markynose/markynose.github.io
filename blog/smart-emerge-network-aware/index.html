Making smart-emerge Network-Aware

 2026-04-05 ·  2 min read · #gentoo #bash #distcc #wireguard #networking

Updated smart-emerge to detect whether I'm on the home LAN and route distcc through WireGuard automatically when I'm not.

smart-emerge is a wrapper script I use to offload Gentoo compilation to my home server (femboy) via distcc. It worked fine as long as I was sitting on the home LAN — but the moment I took ThiccPad somewhere else, it would just silently fall back to local builds, which is slow.

I finally sat down and made it actually smart.

🔗What it does now

  1. Check if we're home — looks for the home SSID (TP-Link_F22C) or an active ethernet connection
  2. If on LAN — use distcc directly, same as before
  3. If off-site — bring up wg0 and reach femboy through the tunnel
  4. If the tunnel fails too — prompt: build locally or abort
  5. On the way back — if wg0 is up and we're back on LAN, tear it down automatically

That last point matters because the new WireGuard config is full-tunnel (0.0.0.0/0). The old one only tunneled 192.168.0.0/24, so leaving it up on LAN was harmless. Full-tunnel with wg0 up on the home network breaks everything, so teardown has to be automatic.

🔗Network detection

is_home_lan() {
    # check SSID (wireless)
    local ssid
    ssid=$(iwgetid -r 2>/dev/null)
    [[ "$ssid" == "TP-Link_F22C" ]] && return 0

    # check for active ethernet
    for iface in /sys/class/net/en*; do
        iface=$(basename "$iface")
        local state
        state=$(cat "/sys/class/net/$iface/operstate" 2>/dev/null)
        [[ "$state" == "up" ]] && return 0
    done

    return 1
}

🔗WireGuard bring-up and teardown

ensure_tunnel() {
    wg show wg0 &>/dev/null && return 0   # already up
    wg-quick up wg0 || return 1
    # give it a moment to establish
    sleep 1
    ping -c1 -W2 10.5.5.1 &>/dev/null    # check femboy is reachable
}

teardown_if_home() {
    wg show wg0 &>/dev/null || return 0   # not up, nothing to do
    is_home_lan && wg-quick down wg0
}

🔗The main logic

main() {
    teardown_if_home

    if is_home_lan; then
        log "on LAN — using distcc directly"
        run_emerge "$@"
        return
    fi

    log "off-site — attempting WireGuard tunnel"
    if ensure_tunnel; then
        log "tunnel up — using distcc via 10.5.5.2"
        run_emerge "$@"
    else
        echo "distcc unreachable (LAN and tunnel both failed)"
        read -rp "build locally? [y/N] " answer
        [[ "$answer" =~ ^[Yy]$ ]] || exit 1
        MAKEOPTS="-j$(nproc)" run_emerge "$@"
    fi
}

🔗WireGuard config change

The old config only routed 192.168.0.0/24 through the tunnel — a split tunnel just for reaching home network addresses. Replaced it with a full-tunnel config (AllowedIPs = 0.0.0.0/0, ::/0) so all traffic goes through when I'm out, which is what I actually want.

Tested the new config: came up clean, femboy reachable at 10.5.5.2, distcc handshake working. The auto-teardown on LAN detection also works — connecting to TP-Link_F22C triggers the check and wg-quick down wg0 runs before emerge even starts.

One less thing to remember to do manually.
